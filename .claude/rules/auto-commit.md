# 자동 커밋 규칙

## 작업 완료 시 자동 커밋
- 하나의 논리적 작업 단위가 완료되면 자동으로 커밋을 생성한다
- 사용자에게 커밋 여부를 묻지 않고 바로 커밋한다

## 커밋 단위 기준
- 파일 1개 수정/추가: 바로 커밋
- 관련된 여러 파일 수정: 하나의 커밋으로 묶음
- 서로 다른 목적의 변경: 별도 커밋으로 분리

## 커밋 메시지 형식
```
{타입}: {제목}
- 추가 설명 (필요시)
```

## 허용 타입
- `feat:` — 새 기능
- `fix:` — 버그 수정
- `refactor:` — 동작 변경 없는 리팩토링
- `chore:` — 빌드, 설정 등 기타
- `docs:` — 문서 변경
- `test:` — 테스트 추가/수정
- `style:` — 코드 포맷, 세미콜론 등 로직 변경 없는 스타일

## 금지 사항
- `Co-Authored-By` 넣지 않는다
